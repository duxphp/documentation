## 管理界面

系统分为前台和后台两部分，前台为网站站点、小程序或App，后台用来管理这些前台的内容，只有有权限的用户登录后才可进行操作。

访问以下地址来登录后台，初次安装后默认账号密码均为 **admin**

```http
http://域名/manage
```


<img
  src="/images/main.png"
/>

登录后您将看到如上界面，界面区域说明：

1. **菜单折叠**，当屏幕空间不足时点击该按钮可折叠左侧菜单。
2. **导航菜单**，后台所有可用功能的菜单导航，安装应用或添加数据魔方后可扩展该菜单，并且菜单和用户权限挂钩。
3. **用户信息**，当前登录的用户信息展示，点击右侧菜单按钮打开用户菜单，可以进行切换明暗风格、退出登录等操作。
4. **功能标题**，当前功能页面的标题信息，如果当前功能含有 Tab 菜单则此处显示 Tab 菜单进行切换。
5. **功能界面**，通过导航菜单跳转到的当前功能操作界面。
6. **操作区域**，如果当前所在界面含有创建、设置等功能则在此处显示。

## 文章管理

系统自带基础应用，用于基本的文章发布，可添加文章多封面图和编辑器内容，编辑器可支持批量图片、视频、附件上传，支持 word 文件直接导入，同时提供粘贴图片的本地化转存功能，为您提供方便的体验， 同时文章可以设置推荐、分类等功能。


## 页面管理

用于管理单个独立的页面，例如个人简介、公司简介、用户协议等独立的文章数据。

## 菜单管理

可以设置多组菜单，用于在不同的页面位置展示不同的菜单，菜单支持树形多级，可以通过模板标签和 Api 接口方便的进行调用，利用该功能可以开发菜单导航、友情链接、功能按钮等功能。

## 数据管理

用于维护系统基本的数据信息，例如省市区数据等其他固定使用的数据。

## 数据魔方

该功能为系统核心功能，又叫自定义模型，可以创建自定义的数据列表和数据发布，合理的使用该功能可以完全实现和文章应用相同的功能，并且数据直接可以互相进行调用，例如内容和分类则为关联内容，该功能主要用于一些 Diy 的需求，可满足大部分的增删差改的开发需求。

## 应用中心

对接 Dux 应用商店，可以通过该应用来进行安装、更新、卸载应用。

## 接口授权

主要针对 Api 接口创建不同的接口密钥，通过该信息的鉴权前端应用可对接系统数据，并且后续将支持接口权限的配置。

## 用户管理

后台角色和用户的管理，角色支持权限设置，一个用户可以绑定多个角色满足不同的权限需求。

## 系统日志

记录后台用户的操作、前台的访问记录等信息，可方便用户操作的溯源与访问统计。

## 站点管理

管理站点信息，设置和配置模板等配置信息。
